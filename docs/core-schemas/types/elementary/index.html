Elementary types

Schema:ethdebug/format/type/elementary→

Elementary types are atomic—they don't contain other types. These form the building blocks that complex types compose.

Numeric types​

Unsigned integers (uint)​

Unsigned integers range from 8 to 256 bits, in increments of 8:

Schema:ethdebug/format/type/elementary/uint
{ "kind": "uint", "bits": 256 }
Schema:ethdebug/format/type/elementary/uint
{ "kind": "uint", "bits": 8 }

The bits field is required and must be a multiple of 8 between 8 and 256.

Signed integers (int)​

Signed integers use two's complement representation:

Schema:ethdebug/format/type/elementary/int
{ "kind": "int", "bits": 256 }

Like uint, the bits field must be a multiple of 8 between 8 and 256.

Fixed-point numbers (ufixed, fixed)​

Fixed-point decimals specify both total bits and decimal places:

Schema:ethdebug/format/type/elementary/ufixed
{ "kind": "ufixed", "bits": 128, "places": 18 }
Schema:ethdebug/format/type/elementary/fixed
{ "kind": "fixed", "bits": 128, "places": 18 }

Address type​

Addresses represent 20-byte Ethereum addresses:

Schema:ethdebug/format/type/elementary/address
{ "kind": "address" }

In Solidity, address payable is distinguished by the payable field:

Schema:ethdebug/format/type/elementary/address
{ "kind": "address", "payable": true }

When payable is omitted, the address's payability is unknown.

Boolean type​

Booleans represent true/false values:

Schema:ethdebug/format/type/elementary/bool
{ "kind": "bool" }

In the EVM, booleans occupy a full 32-byte word where 0 is false and any non-zero value is true.

Byte types​

Fixed-size bytes (bytes)​

Fixed-size byte arrays range from 1 to 32 bytes:

Schema:ethdebug/format/type/elementary/bytes
{ "kind": "bytes", "size": 32 }
Schema:ethdebug/format/type/elementary/bytes
{ "kind": "bytes", "size": 4 }

The size field is required for fixed-size bytes.

Dynamic bytes (bytes)​

Dynamic byte arrays have no size limit:

Schema:ethdebug/format/type/elementary/bytes
{ "kind": "bytes" }

When size is omitted, the type represents dynamically-sized bytes.

String type​

Strings represent UTF-8 encoded text:

Schema:ethdebug/format/type/elementary/string
{ "kind": "string" }

Strings are dynamically sized. Languages that treat strings as character arrays may choose to represent them as array types instead.

Enum type​

Enums represent a fixed set of named values:

Schema:ethdebug/format/type/elementary/enum
{
  "kind": "enum",
  "definition": {
    "name": "Status"
  },
  "values": ["Pending", "Active", "Completed"]
}

The values field lists all possible values in order. The underlying representation is typically a uint8 (or larger if needed).

Contract type​

Contract types represent references to contract instances:

Schema:ethdebug/format/type/elementary/contract
{ "kind": "contract" }

Contract types may include a definition field when the contract is defined in source code:

Schema:ethdebug/format/type/elementary/contract
{
  "kind": "contract",
  "definition": {
    "name": "MyToken"
  }
}

In the EVM, contract references are addresses (20 bytes), but the contract type carries additional semantic information about which contract interface applies.

User-defined types with definitions​

Some elementary types (particularly enums) are defined in source code. These include a definition field:

Enum with source locationSchema:ethdebug/format/type/elementary/enum
{
  "kind": "enum",
  "definition": {
    "name": "OrderStatus",
    "location": {
      "source": { "id": 42 },
      "range": {
        "offset": 150,
        "length": 85
      }
    }
  },
  "values": ["Created", "Filled", "Cancelled"]
}

The definition field can include:

  • name: The identifier used in source code
  • location: A source range where the type is defined
