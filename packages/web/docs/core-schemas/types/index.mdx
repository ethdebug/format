# Types

<SpecLink schema="ethdebug/format/type" href="/spec/type/overview" />

Types in **ethdebug/format** describe the structure of data — what shape it
takes and how its parts relate. Types don't specify where data lives; that's
what [pointers](/docs/core-schemas/pointers) do. Together, types and pointers give debuggers
everything needed to locate bytes and interpret them as meaningful values.

For the mental model behind types, see [Concepts: Types](/docs/concepts/types).
